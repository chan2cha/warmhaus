# Operations Guide

## End-to-End Flow

### 1. 고객 접수

고객은 공개 페이지에서 문의 폼으로 진입합니다. 폼에는 기본 정보뿐 아니라 공사 범위, 예산, 가구 요청, 상담 방식, 희망 시간대가 포함됩니다.

### 2. 리드 생성

폼 제출 시 `leads` 레코드가 생성됩니다. 동시에 `appointments`에 협의 상태의 일정 레코드가 만들어지고, `appointment_candidates`에 희망 시간 후보가 저장됩니다.

### 3. 운영자 확인

운영자는 관리자 인박스에서 신규 리드를 확인합니다. 일정 기준으로 오늘, 내일, 이번 주, 미예약 탭으로 나누어 볼 수 있습니다.

### 4. 고객 응대

리드 상세 화면에서 운영자는 고객 희망 시간 후보를 검토합니다. 후보별로 상담 제안 문자 또는 확정 문자 템플릿을 생성할 수 있습니다.

### 5. 상태 반영

후보는 다음 흐름으로 전이됩니다.

```text
PROPOSED -> PENDING -> CUSTOMER_CONFIRMED -> CONFIRMED
PROPOSED -> PENDING -> CUSTOMER_DECLINED
```

확정된 일정은 이후 아래 상태로 이동할 수 있습니다.

```text
CONFIRMED -> RESCHEDULE_REQUESTED
CONFIRMED -> CANCELED
```

## Rule and Notice Operations

### Notice

- 고객 폼 상단에 노출되는 문구 관리
- 전화번호 및 추가 안내 문구 편집

### Rules

- 접수 가능 지역 관리
- 마감 월 관리
- 특정 날짜 부분 가능 관리
- 최소 / 선호 예산 기준 관리

## Current Operational Risks

### 1. 문자 발송 자동화 미구현

현재는 문자 API 직접 발송이 아니라, 템플릿 생성 후 복사 또는 `sms:` 링크 기반으로 처리됩니다.

### 2. 캘린더 페이지 미구현

관리자 인박스에서 캘린더 링크가 보이지만 실제 페이지는 아직 없습니다.

### 3. 첨부파일 미연결

폼에서 파일을 선택할 수 있으나 스토리지 업로드는 구현되어 있지 않습니다.

### 4. 인증 일관성 부족

관리자 설정 API 보호 수준이 일관되지 않아 운영 전 보강이 필요합니다.

### 5. Rule 정합성 점검 필요

운영 Rule 저장/조회와 리드 등급 계산 로직 간 키 사용이 완전히 정리되어 있는지 확인이 필요합니다.
